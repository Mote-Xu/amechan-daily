# 项目总结 — 超天酱日常推文小站

> v3.2 — 架构稳定，转向内容质量。
> 最后更新：2026-06-16

---

## 当前状态

纯前端 HTML/CSS/JS + Python http.server 极简后端。DeepSeek V4-pro。

### 已解决（架构层面）
- F7 戳一戳：水位线后台补池，O(1) 释放
- JINE 聊天：微聚合窗口 (300-1200ms) + 打断重发 + 回复计数防死循环
- 贴图标签发送实际标签名 `[好][嘤嘤]`
- 后端无状态，前端单源存储
- 删除全部硬编码触发词，100% AI 回复

### 残余
- 弹幕 CSS 偶尔消失
- 刷新后偶发发不了消息

---

## 🔴 当前主要诉求：内容质量

架构已稳，现在需要让 AI 产出的**内容本身**更有趣、更有"活人味"。

### 需要强化的三个方向

**1. JINE 聊天 — 糖糖风格**

核心 prompt 文件：`prompts.py`
- `JINE_REPLY_SYSTEM`（贴图回复）
- `JINE_TEXT_REPLY_SYSTEM`（文字回复）
- `AME_JINE_PERSONA`（人设）

当前问题：
- 回复有时偏平淡、偏"标准傲娇模板"
- 缺乏游戏原版那种"攻击性可爱"和"黏人/身体依赖"的独特混合
- 需要更多具体的行为指导而非抽象性格描述

**2. 推博 (Poketter) — 超天酱 + 糖糖**

核心 prompt 文件：`prompts.py`
- `KANGEL_PERSONA`（超天酱人设）
- `AME_DIARY_PERSONA`（糖糖日记人设）
- `get_timeline_prompt()`（时间线生成）
- `EVENT_POOL`（事件池）

当前问题：
- 事件池有些空泛（"直播""天气""粉丝互动"）
- 推文有时缺乏具体细节和故事性
- 超天酱和糖糖的反差感可以更强

**3. 弹幕**

核心文件：
- `static/index.html` — `DANMAKU_OEN`（应援）/ `DANMAKU_TUCAO`（吐槽）
- `prompts.py` — `EVENT_POOL`（影响时间线内容，间接影响弹幕氛围）

当前问题：
- 文案池偏小，重复率高
- 缺少当下网络流行语和梗
