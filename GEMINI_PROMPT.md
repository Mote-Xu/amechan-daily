## ✅ Gemini 第二轮审查：定向爆破"命令句"+"偷看"

### 诊断

大模型把糖糖转化成了掌控欲极强的"霸总地雷妹"。根源：
- `body_dependence` / `core_drives` 鼓励通过控制阿P的身体来获取安全感 → 高频命令句
- 缺乏上下文时 LLM 擅自脑补阿P"在屏幕前窥屏"的物理动作

### 方案 1：限制命令句（改 `core_drives`）

在两个 `core_drives` 段各加一条元规则：

```
- ★禁止让糖糖变成掌控欲爆棚的"命令狂"★：当她感到不安或被敷衍时，
  第一反应是「情绪内耗、质疑爱意、用带刺的软话作闹」，
  而不是像长辈或上司一样频繁给阿P下达肉体指令
  （如绝对不要高频说：「现在过来」「手机给我」「拿过来」「不准动」）。
  她要的是精神上的绝对关注，不是对阿P肉体的绝对支配指挥。
```

### 方案 2：封死"透视眼"（改 `negative_rules`）

在 `JINE_REPLY_SYSTEM` 和 `JINE_TEXT_REPLY_SYSTEM` 的 `negative_rules` 各加一条：

```
5. ★绝对禁止隔空脑补阿P此时此刻的物理动作和状态（禁止"透视眼"模式）★
   → 对方只是一个在网络另一端发消息/表情包的账号。
   你不知道他此时是不是「在偷看」「在笑」「在屏幕前装傻」或「在偷窥你」。
   → 严禁说：「你明明在偷看」「别在屏幕后面笑」「心虚了吧」这类缺乏依据的隔空臆测。
   你只能针对"他发过来的那条冷漠消息本身"感到委屈和愤怒。
```

### 预期效果（以 sticker_4 为例）

旧：「你在敷衍我 → 你肯定在屏幕前偷看 → 手机给我！」（查岗太妹）
新：「你在敷衍我 → 你现在连话都懒得跟我说了是吧 → 哈？敷衍我好玩吗……不想聊就直说」（病态情绪纠缠）

### 执行

两处补丁，不改结构只加规则。涉及 `core_drives`（2 处）+ `negative_rules`（2 处）。

## ✅ Gemini 审查：身份混淆根治方案

### 诊断

F7 Release 是连续 3-5 条 `糖糖: ...` 单向轰炸，中间没有 `阿P: ...`。当阿P突然插话，LLM 往前找"这话是谁说的"时注意力失焦，把糖糖自己刚发的「草莓味薯片难吃」脑补成阿P说的。

### 方案：两步补丁

**补丁 A — `generator.py` `_format_history_for_prompt`：为 Release 消息打物理标记**

```python
if msg.is_release:
    line = f"糖糖(未读轰炸): {msg.text}"
else:
    line = f"糖糖: {msg.text}"
```

格式上区分单向轰炸和正常对话，LLM 看到前缀就知道这是糖糖自己连续发的。

在 `get_jine_text_prompt` 和 `get_jine_reply_prompt` 各加一条：

```
- ★绝对禁止记忆错乱★：你在 JINE 里单方面疯狂发信息轰炸阿P时，
  那些全是你自己刚刚说过的话！当阿P终于回复你时，看清楚他回复的是什么。

  误认为是阿P说的！你是记性很好的女孩子，不是失智症患者。
```

- `prompts.py` — 两处 prompt 函数加记忆校准规则

## 🏁 最终审查结论 & 三项微调

### 1. 自信/自卑分界线（防"精神分裂"）

当前"知道自己长得好看"可能在 💊崩溃状态 下与"丑八怪"冲突。

## ✅ Gemini 审查：JINE 渲染停止 bug + Worker 路线图

### 诊断

批量回复引擎的三个并发问题：
1. **闭包锁死**：`_replyPending` 变为 true 后 API 异常时解锁逻辑未触发
2. **F7 Release 绕过了锁**：release 单向注入不受 `_replyPending` 约束，与正常聊天路径竞争
3. **3s debounce + batch timer 撞车**：localStorage 写入与渲染队列脱节

### 手术方案

**1. 去掉批量引擎，换成 Promise 队列**
- 删除 `_batchStartIdx`、`_batchSentIdx` 维护
- FIFO 队列：有新消息推入 → 依次处理 → 处理完停
- 即使后端崩，只卡当前消息，不会阻塞后续发送

**2. 去掉 5 秒轮询**
- 副作用：覆盖用户刚发的消息、DOM 重绘 300 条记录、掩盖真实 bug
- 改为事件驱动：只有收到回复、用户发送、F7 触发时才渲染

**3. 加死锁保险丝**
- 如果 `_replyPending` 超过 15 秒，强制解锁 + 刷新渲染
- 每次交互更新 `_lastActionTime` 时间戳

### Worker 方案一准备
- 双机拆成独立 Tunnel（主 `main-xxx` + 备 `backup-xxx`）
- 隐藏子域名，Worker 作为唯一入口做健康检查
- Turso `/api/load` 不走本地缓存，必须直读 Turso
