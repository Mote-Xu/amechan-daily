## 一句话概述

模拟《主播女孩重度依赖》中超天酱社交账号的轻量 Web 应用，AI 自动以超天酱（和其真实人格糖糖）的语气发推文，展示在粉紫像素风 feed 页面上。使用游戏原版素材（自拍头像、UI 图标），还原 Poketter 质感。

## 项目设计

### 核心机制 — 三层内容体系

| 层级 | 角色 | 语气 | 对象 |
|------|------|------|------|
| **L1: Poketter** | 超天酱 | 元气、emoji、撒娇，偶尔漏不安 | 对粉丝 |
| **L2: 糖糖日记** | 糖糖 | 阴暗、疲惫、碎句、自救 | 自言自语 |
| **L3: JINE 私信** 🔜 | 糖糖 → 阿P | 求救、依赖、怨念、偶尔温柔 | 对阿P |

三层递进——从公开假面 → 私密碎碎念 → 亲密关系中的求救，增强窥探沉浸感。

### 技术方案

| 项 | 选型 |
|------|------|
| LLM | DeepSeek API (deepseek-v4-pro) |
| 前端 | 纯 HTML/CSS/JS（Poketter v1.1，含 CRT + Win95 + 像素风） |
| 后端 | Python `http.server`（内置模块） |
| 存储 | 本地 JSON 文件 |
| 游戏素材 | UnityPy 从 Steam 版提取（52 张自拍 + 30+ UI 图标 + 400+ FanArts） |
| 环境 | WSL2, Python 3.11, conda: deepseek_v4_api |

### 当前卡点
当前界面仍偏静态——只有 feed 滚动 + 状态栏 + 图标。缺少「此刻正在发生」的动态感和「多个 App 切换」的桌面模拟感。F5-F7 应该能解决这个问题。头像/自拍已经分清了，接下来核心是让页面「动」起来。

## 后续问题（留给 Gemini）

1. **JINE 私信内容风格**：即时通讯相比推文，语气应该更碎、更即兴、更不设防。如何让 LLM 产出「像是匆忙打出来又删了一半」的感觉？
2. **直播组件的视觉节奏**：如果只放一个 LIVE 角标 + 静态画面会觉得假。如何在不动用大量游戏动画帧的前提下，营造「正在直播」的临场感？
3. **三层内容的时间关联**：Poketter 推文、糖糖日记、JINE 私信——三者的时间线和内容是否需要有时序关联？（例如：先发了元气推文，隔了几分钟糖糖在 JINE 里对阿P说「今天好累」）
4. **GitHub Pages 部署后的内容更新**：前端纯静态好部署，但推文生成依赖 Python 后端。如何做到部署后仍能自动更新内容？GitHub Actions 定时触发脚本 + 提交 JSON 到 repo？
