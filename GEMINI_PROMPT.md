# 项目总结 — 超天酱日常推文小站

> v3.0 — 微聚合回复引擎。
> 最后更新：2026-06-16

---

## 当前架构

纯前端 HTML/CSS/JS + Python `http.server` 极简后端。DeepSeek V4-pro API。

## F7 戳一戳 ✅

水位线后台补池。`pop_from_pool()` 释放后剩余对 < 2 → 后台线程静默生成。前端 O(1) 单 POST。

## JINE 聊天回复引擎 v3 ✅

**核心原则**：像真人微信聊天，不是 AI 问答。

### 微聚合窗口
- 用户发消息 → 开始 300-1200ms 随机窗口
- 持续输入时不断刷新窗口
- 停止输入后收集**所有未回复消息** → 一次 API 请求
- 回复插入聊天末尾

```
t=0s   [哭]      → 窗口开始
t=0.3s [趴]      → 窗口刷新
t=0.7s [打滚]    → 窗口刷新
t=1.2s 停止      → 窗口到期 → 一次 API:
                 AI 看到 [哭][趴][打滚]
                 回复: "今天怎么这么惨" + [拍拍猫]
```

### 打断重发
API 生成期间用户继续发消息 → abort 当前请求 → 重新开窗 → 合并新旧消息重发

### 贴图+文字分离
`[stamp:xxx]` 前端后端双解析。后端白名单过滤 → `ame_sticker` 字段。前端渲染时再次解析兜底。

### 后端
- `/api/jine/chat` 统一无状态端点
- `generate_jine_chat(text, sticker, history)` — 接收前端完整 history
- 文字温度 0.7，贴图温度 0.85

## 残余问题

1. 弹幕偶尔消失
2. 刷新后偶发发不了消息
