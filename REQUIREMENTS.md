| ID | 功能 | 目标 | 状态 |

## JINE 聊天回复策略（v3）

**核心原则**：像真人微信聊天，不是 ChatGPT 问答。

1. **微聚合窗口**：300-1200ms 随机窗口，持续输入时不断刷新，停止后一次 API 请求
2. **上下文回复**：AI 看到的是聚合后的整体，可以忽略部分消息、只回整体氛围
3. **打断重发**：API 生成期间有新输入 → abort 当前请求 → 重新开窗合并
4. **高频保护**：短时间大量消息仍只生成一次回复，不逐条对应
5. **贴图+文字分离**：`[stamp:xxx]` 前端后端双解析，文字归文字、贴图归贴图

## 已解决

- 戳一戳 pool 竞态 → 水位线后台补池
- JINE 延迟 → 耗时重叠 + 微聚合窗口
- 对话感泄漏 → 温度 0.7 + prompt 收紧
- JINE 状态不同步 → 后端无状态
- 逐条回复僵硬 → v3 聚合窗口
